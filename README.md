# Finance Application

## Project Structure

```
├── src/                    # Main application source
│   ├── components/         # Reusable UI components
│   ├── layouts/            # Page layouts
│   ├── pages/              # Application pages
│   ├── assets/             # Static resources (images, fonts, etc.)
│   ├── styles/             # Global CSS files and variables
│   ├── utils/              # Helper functions
│   └── services/           # Application services and API integration
├── public/                 # Public files
└── docs/                   # Project documentation
```

## Core Components

- **components**: Contains reusable UI components such as buttons, forms, and cards
- **layouts**: Shared page layouts
- **pages**: Main application pages
- **assets**: Static resources like images and fonts
- **styles**: Centralized CSS files and variable definitions
- **utils**: Shared helper functions
- **services**: Business logic and API services

## Design System

The application uses a unified design system that includes:

- Harmonious colors defined as CSS variables
- Specified font families
- Reusable UI components
- Consistent spacing and styles

## Development Guidelines

1. Follow the specified folder structure for code organization
2. Use shared components from the components folder
3. Maintain design consistency using defined CSS variables
4. Document new components and functions
